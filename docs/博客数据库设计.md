# 个人博客项目数据库设计

## 📋 项目概述

本文档为个人博客项目的数据库设计方案，基于Next.js 16框架，采用PostgreSQL作为主数据库。设计核心原则：

- **文件存储优先**：文章内容存储在服务端文件系统，数据库仅存储元数据和URL
- **访客友好**：支持访客评论，无需完整用户注册系统
- **性能优化**：合理的索引设计和查询优化
- **扩展性强**：支持未来功能扩展

---

## 🎯 设计理念

### 1. 存储策略
- **文章文件**：`content/articles/` 目录，文件名使用slug
- **图片媒体**：`uploads/images/` 目录，按年月分目录存储
- **数据库职责**：存储元数据、关系映射、URL路径

### 2. 功能模块
- 文章管理系统（分类、标签、搜索）
- 评论系统（支持嵌套回复）
- AI作品展示
- 媒体文件管理（图片上传、存储、关联）
- 文章媒体关联管理（支持文章删除时的图片清理）
- SEO优化支持

## 📊 数据库表结构设计

### 0. 预置定义（枚举类型与触发器）

在创建表之前，需要先定义枚举类型和自动更新时间戳的触发器函数：

```sql
-- ========================================
-- 枚举类型定义
-- ========================================
CREATE TYPE article_status AS ENUM ('draft', 'published', 'archived');
CREATE TYPE comment_status AS ENUM ('pending', 'approved', 'spam', 'trash');
CREATE TYPE media_usage_type AS ENUM ('article', 'avatar', 'artwork', 'general');
CREATE TYPE article_media_usage AS ENUM ('thumbnail', 'attachment');
CREATE TYPE artwork_media_usage AS ENUM ('main', 'thumbnail', 'process', 'variant');
CREATE TYPE artwork_status AS ENUM ('draft', 'published');
CREATE TYPE setting_type AS ENUM ('string', 'number', 'boolean', 'json');

-- ========================================
-- 自动更新 updated_at 的触发器函数
-- ========================================
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### 1. 文章系统核心表

#### 1.1 文章表 (articles)
```sql
CREATE TABLE articles (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    slug VARCHAR(255) UNIQUE NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    excerpt TEXT,
    thumbnail_url VARCHAR(500),
    author_name VARCHAR(100) DEFAULT '木心',
    reading_time INTEGER DEFAULT 0,
    view_count INTEGER DEFAULT 0,
    status article_status DEFAULT 'published',
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引

CREATE INDEX idx_articles_slug ON articles(slug);
CREATE INDEX idx_articles_author ON articles(author_name);
CREATE INDEX idx_articles_created_at ON articles(created_at DESC);

-- 自动更新 updated_at 触发器
CREATE TRIGGER trigger_articles_updated_at

    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();

-- 表和字段注释
COMMENT ON TABLE articles IS '文章主表';
```

### 4. 后台管理系统

#### 4.1 管理员用户表 (admin_users)

**设计说明：**
- 单管理员模式，无需复杂的用户注册系统
- 仅用于后台管理员登录认证
- 集成 JWT Token 会话管理
- 包含登录失败锁定机制，防止暴力破解

```sql
-- ========================================
-- 管理员枚举类型定义
-- ========================================
CREATE TYPE admin_status AS ENUM ('active', 'inactive', 'locked');

-- ========================================
-- 管理员用户表
-- ========================================
CREATE TABLE admin_users (
    id SERIAL PRIMARY KEY,

    -- 基本信息
    username VARCHAR(100) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    display_name VARCHAR(100),
    avatar_url VARCHAR(500),

    -- 认证信息
    password_hash VARCHAR(255) NOT NULL,
    password_salt VARCHAR(255) NOT NULL,

    -- 账户状态
    status admin_status DEFAULT 'active',

    -- 安全追踪
    last_login_at TIMESTAMP,
    last_login_ip VARCHAR(45),
    login_attempts INTEGER DEFAULT 0,
    locked_until TIMESTAMP,

    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_admin_users_username ON admin_users(username);
CREATE INDEX idx_admin_users_email ON admin_users(email);
CREATE INDEX idx_admin_users_status ON admin_users(status);
CREATE INDEX idx_admin_users_last_login_ip ON admin_users(last_login_ip);

-- 自动更新 updated_at 触发器
CREATE TRIGGER trigger_admin_users_updated_at
    BEFORE UPDATE ON admin_users
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();

-- 表和字段注释
COMMENT ON TABLE admin_users IS '后台管理员用户表（单管理员模式）';
COMMENT ON COLUMN admin_users.username IS '登录用户名，唯一标识';
COMMENT ON COLUMN admin_users.email IS '邮箱地址，用于密码重置和通知';
COMMENT ON COLUMN admin_users.display_name IS '后台显示名称';
COMMENT ON COLUMN admin_users.avatar_url IS '管理员头像URL';
COMMENT ON COLUMN admin_users.password_hash IS '密码哈希值（使用MD5算法）';
COMMENT ON COLUMN admin_users.password_salt IS '密码盐值，增强安全性';
COMMENT ON COLUMN admin_users.status IS '账户状态：active（活跃）/inactive（停用）/locked（锁定）';
COMMENT ON COLUMN admin_users.last_login_at IS '最后登录时间，用于审计';
COMMENT ON COLUMN admin_users.last_login_ip IS '最后登录的IP地址，用于异常检测';
COMMENT ON COLUMN admin_users.login_attempts IS '连续登录失败次数，超过5次自动锁定';
COMMENT ON COLUMN admin_users.locked_until IS '账户被锁定的过期时间';
COMMENT ON COLUMN admin_users.created_at IS '账户创建时间';
COMMENT ON COLUMN admin_users.updated_at IS '账户最后修改时间';
```

**字段说明：**
- `password_hash`: 使用 MD5 算法加密存储
- `password_salt`: 密码盐值，与 MD5 配合使用增强安全性
- `login_attempts`: 记录连续登录失败次数
  - 每次登录失败 +1
  - 登录成功重置为 0
  - ≥5 次时自动锁定账户
- `locked_until`: 账户锁定到期时间
  - 登录失败 5 次后设置为当前时间 + 15 分钟
  - 到期后自动解锁
  - 登录时需检查 `locked_until IS NULL OR locked_until < CURRENT_TIMESTAMP`
- `last_login_ip`: 支持 IPv4 和 IPv6 格式（最长 45 字符）

**安全特性：**
1. **密码加密**：使用 MD5 算法，带盐值存储
2. **登录锁定**：5 次失败后锁定 15 分钟
3. **IP 追踪**：记录登录 IP，用于异常检测
4. **JWT 集成**：配合项目现有的 `jwt.ts` 工具进行会话管理

```sql
);

-- 索引
CREATE INDEX idx_artwork_media_artwork_id ON artwork_media(artwork_id);

COMMENT ON COLUMN artwork_media.created_at IS '关联创建时间';
```

**字段说明：**
- `usage_type`: 标识图片的使用场景
  - `main`: 主展示图（在画廊中显示的主图）
  - `thumbnail`: 缩略图（列表展示使用）

- `ON DELETE CASCADE`: 删除作品时自动删除关联记录
- `ON DELETE RESTRICT`: 防止误删除仍在使用的媒体文件

**使用场景：**
1. AI作品上传时，关联主图和缩略图
2. 记录AI创作过程的多个阶段图片
3. 保存同一提示词生成的多个变体
4. 作品删除时，自动清理不再使用的图片

## 🔧 性能优化建议

### 1. 索引策略
- **复合索引**：为常用查询组合建立复合索引
- **覆盖索引**：包含查询所需的所有字段，避免回表
- **定期维护**：定期分析索引使用情况，清理无用索引

### 2. 查询优化
```sql
-- 首页文章列表查询（已发布，按时间排序，获取主分类）
SELECT a.id, a.title, a.slug, a.excerpt, a.thumbnail_url,
       a.author_name, a.reading_time, a.view_count, a.published_at,
       c.name as primary_category_name, c.color as primary_category_color
FROM articles a
LEFT JOIN article_categories ac ON a.id = ac.article_id AND ac.is_primary = TRUE
LEFT JOIN categories c ON ac.category_id = c.id
WHERE a.status = 'published'
ORDER BY a.published_at DESC

UPDATE categories c

-- 步骤1: 查询作品关联的所有图片ID
SELECT DISTINCT awm.media_id
FROM artwork_media awm
WHERE awm.artwork_id = 1;

-- 步骤2: 删除作品（会自动删除artwork_media中的关联记录，因为ON DELETE CASCADE）

-- 步骤3: 清理不再被使用的孤立媒体文件
DELETE FROM media_files
WHERE id NOT IN (
    SELECT DISTINCT media_id FROM article_media
    UNION
    SELECT DISTINCT media_id FROM artwork_media
)
AND usage_type = 'general';

-- 画廊页面：获取已发布的AI作品及其主图
SELECT a.id, a.title, a.slug, a.description,
       a.view_count, a.like_count,
       m.file_url as main_image_url,
       mt.file_url as thumbnail_url
FROM ai_artworks a
LEFT JOIN artwork_media awm_main ON a.id = awm_main.artwork_id AND awm_main.usage_type = 'main'
LEFT JOIN media_files m ON awm_main.media_id = m.id
LEFT JOIN artwork_media awm_thumb ON a.id = awm_thumb.artwork_id AND awm_thumb.usage_type = 'thumbnail'
LEFT JOIN media_files mt ON awm_thumb.media_id = mt.id
WHERE a.status = 'published'
ORDER BY a.is_featured DESC, a.sort_order, a.created_at DESC
LIMIT 20;

-- 统计所有媒体文件的使用情况（包括文章和作品）
SELECT m.id, m.stored_name, m.file_url,
       COALESCE(article_count, 0) as article_usage,
       COALESCE(artwork_count, 0) as artwork_usage,
       COALESCE(article_count, 0) + COALESCE(artwork_count, 0) as total_usage
FROM media_files m
LEFT JOIN (
    SELECT media_id, COUNT(*) as article_count
    FROM article_media
    GROUP BY media_id
) am ON m.id = am.media_id
LEFT JOIN (
    SELECT media_id, COUNT(*) as artwork_count
    FROM artwork_media
    GROUP BY media_id
) awm ON m.id = awm.media_id
ORDER BY total_usage DESC;

-- 管理员登录验证（查询用户名和状态）
SELECT id, username, password_hash, password_salt, status,
       login_attempts, locked_until
FROM admin_users
WHERE username = $1 AND status = 'active';

-- 管理员登录成功：更新登录信息和重置失败次数
UPDATE admin_users
SET last_login_at = CURRENT_TIMESTAMP,
    last_login_ip = $1,
    login_attempts = 0,
    locked_until = NULL
WHERE id = $2;

-- 管理员登录失败：增加失败次数，检查是否需要锁定
UPDATE admin_users
SET login_attempts = login_attempts + 1,
    locked_until = CASE
        WHEN login_attempts >= 4 THEN CURRENT_TIMESTAMP + INTERVAL '15 minutes'
        ELSE locked_until
    END
WHERE id = $1
RETURNING login_attempts, locked_until;

-- 检查账户是否被锁定
SELECT id, username, locked_until,
       (locked_until IS NULL OR locked_until < CURRENT_TIMESTAMP) as is_unlocked
FROM admin_users
WHERE id = $1;

-- 获取管理员信息（用于JWT payload）
SELECT id, username, email, display_name, status
FROM admin_users
WHERE id = $1 AND status = 'active';
```

### 4. 分区策略（数据量大时）
```sql
-- 按年月分区评论表（适用于数据量大的情况）
-- PostgreSQL 原生分区语法
CREATE TABLE comments_partitioned (
    id BIGSERIAL,
    article_id BIGINT NOT NULL,
    parent_id BIGINT,
    author_name VARCHAR(100) NOT NULL,
    content TEXT NOT NULL,
    status comment_status DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id, created_at)
) PARTITION BY RANGE (created_at);

-- 创建分区
```

## 🛡️ 数据安全建议

### 1. 备份策略
- **定期全量备份**：每天凌晨进行全量备份
- **增量备份**：每小时进行增量备份
- **异地备份**：备份文件存储在不同地理位置

### 2. 访问控制
- **最小权限原则**：应用账户只给必要的数据库权限
- **连接加密**：使用SSL/TLS加密数据库连接
- **IP白名单**：限制数据库访问IP范围

### 3. 数据验证
```sql
-- 评论内容过滤（防SQL注入）
-- PostgreSQL 使用预处理语句
PREPARE insert_comment (BIGINT, TEXT) AS
    INSERT INTO comments (article_id, content) VALUES ($1, $2);
EXECUTE insert_comment(1, '用户的评论内容');
DEALLOCATE insert_comment;

-- 邮箱格式验证（使用 PostgreSQL 正则表达式语法）
UPDATE comments
SET author_email = NULL
WHERE author_email IS NOT NULL
AND author_email !~ '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$';
```

### 4. 管理员账户安全
- **密码加密**：使用 MD5 算法加密密码，配合盐值增强安全性
- **登录锁定**：连续 5 次登录失败后，账户锁定 15 分钟
- **IP 追踪**：记录每次登录的 IP 地址，用于异常检测
- **会话管理**：使用 JWT Token 管理会话，配合 `login_attempts` 和 `locked_until` 字段实现自动锁定/解锁
- **密码强度**：强制使用强密码（至少 8 位，包含大小写字母和数字）
- **定期更换**：建议每 90 天更换一次管理员密码
